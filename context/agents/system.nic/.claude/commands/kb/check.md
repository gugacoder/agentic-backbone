Valida se um approach é consistente com decisões anteriores documentadas no KB.

Passos:
1. Leia `kb/HOME.md` para entender o índice do KB
2. Leia os artigos relevantes em `kb/atlas/concepts/` e `kb/atlas/connections/`
3. O argumento descreve o approach que o usuário quer validar (ex: "usar SQLite para sessions")
4. Verifique se existem decisões, padrões ou lições no KB que suportem ou contradigam esse approach
5. Reporte: consistente, inconsistente, ou sem dados suficientes — com referências via [[wikilinks]]

Se nenhum argumento for fornecido, peça ao usuário qual approach quer validar.
